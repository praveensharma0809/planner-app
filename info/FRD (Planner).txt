Planner System — Feature Requirements Document (FRD)
1. Product Overview

The Planner System enables students preparing for competitive exams to generate and manage a structured study plan. The system collects syllabus structure, planning parameters, and global constraints to automatically generate an optimized day-wise study schedule that the user can visually inspect and modify before committing.

Primary goal: Convert syllabus + time constraints into a structured executable study plan.

2. Core Objectives
Objective	Description
Structured Planning	Allow users to structure their syllabus hierarchy
Automated Plan Generation	Generate optimized day-wise study plan
Visual Management	Enable users to modify plan visually
Constraint Handling	Respect deadlines, dependencies, and availability
Minimal User Effort	Collect only necessary planning inputs
3. System Workflow
Phase	Name	Purpose
Phase-1	Subject Structure	Build syllabus dataset
Phase-2	Topic Planning Inputs	Define workload and constraints
Phase-3	Global Planning Constraints	Define time environment
Phase-4	Plan Generation & Editing	Generate and allow modification
Phase-5	Final Confirmation	Commit finalized plan
4. Phase-1 — Subject Structure Builder
Purpose

Allow users to define their syllabus hierarchy.

Interface

Grid / spreadsheet-like interface.

Fields
Field	Required	Description
Subject Name	Yes	Top-level syllabus unit
Topic / Chapter	Optional	Subdivision of subject
Subtopic	Optional	Smaller planning unit
Manual Ordering	Optional	Drag-drop ordering
User Actions
Action	Behavior
Add Subject	Creates new top-level category
Add Topic	Adds item under subject
Add Subtopic	Adds nested topic
Drag-Drop	Reorder structure
Delete	Remove item
Output Dataset
Subject
 ├ Topic
 │   ├ Subtopic
5. Phase-2 — Topic Planning Inputs
Purpose

Collect planning parameters for each syllabus unit.

Interface

Editable table linked to Phase-1 topics.

Fields
Field	Required	Description
Expected Time Required	Yes	Total time needed for topic
Topic Priority	Optional	High / Medium / Low
Earliest Start Date	Optional	Topic cannot start before this date
Topic Deadline	Optional	Desired completion date
Topic Dependency	Optional	Requires another topic completion
Practice Sessions Required	Optional	Number of practice sessions
Avg Practice Session Time	Conditional	Asked if practice sessions filled
Revision Sessions Required	Optional	Number of revision sessions
Avg Revision Session Time	Conditional	Asked if revision sessions filled
Behavior

Topics from Phase-1 automatically populate this table.

9. Core Planning Algorithm

The planner engine generates schedules based on:

topic effort
topic priority
topic dependencies
topic deadlines
daily capacity
availability constraints

Scheduling strategy:

priority sorting
deadline adjustment
greedy allocation
capacity balancing
weekly distribution
10. Edge Case Handling
Scenario	System Response
Topic effort exceeds daily capacity	Split across days
Total workload exceeds available capacity	Warn user
Deadline impossible	Raise deadline conflict
Dependency loop	Prevent scheduling
Topic without effort	Ignore until defined
