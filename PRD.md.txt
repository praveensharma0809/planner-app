5. Technical Architecture
5.1 Stack

Next.js 14 (App Router)

Supabase (Postgres + Auth + RLS)

TypeScript

TailwindCSS

Server Actions

Supabase SSR (@supabase/ssr)

Middleware session refresh

5.2 Authentication Model

Cookie-based session storage

SSR Supabase client

Middleware refreshes tokens

Server Actions read cookies

All mutations run server-side

RLS enforced via auth.uid()

Security Principle:

No client-side mutation logic.
